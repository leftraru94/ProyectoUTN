Arquitectura del Sistema

  • Recepción (Quiosco): Registra alumnos y genera credenciales.
  • Alumno (Móvil QR): Carga su perfil y consulta estado (notas/deuda).
  • Administración: Carga las notas y deudas en la base central.
  • Base de Datos (API): Google Sheets conecta todo.

Hoy construiremos el ecosistema completo: Dos Apps Frontends consumiendo la misma API en la red.

Actividad 1: Conexión Física P2P

Conectaremos directamente nuestras dos áreas principales usando una topología Punto a Punto.

  • PC 1: Computadora de la Recepcionista.
  • PC 2: Nodo Central (Administración).
  • Tarea: Conectar un único cable de red Ethernet uniendo ambas computadoras.
💻 ↔️ 💻

Actividad 2: IPs Estáticas para la Red Local

Para que ambas PCs se vean directamente, necesitan estar en la misma subred.

PC 1 (Recepción):
IP: 192.168.1.10 - Máscara: 255.255.255.0

PC 2 (Nodo Central):
IP: 192.168.1.11 - Máscara: 255.255.255.0

Prueba: Abre la terminal (CMD) en la PC 1 y ejecuta ping 192.168.1.11. Si responde, ¡hay red!

Actividad 3: La API Central en Google Sheets

  • Abre la carpeta api-drive/ y sigue los pasos del archivo instrucciones.md.
  • Crea un Sheets con las columnas: Fecha, Nombre, Usuario, Password, DatosExtra, Notas, Inasistencias, Deuda.
  • Ve a Extensiones > Apps Script.
  • Pega el código de api-drive/codigo.gs (Maneja el alta desde recepción y las consultas del alumno).
  • Implementar > Nueva implementación > Aplicación Web (Cualquier persona).
  • ¡Copia la URL de la API!

Actividad 4: Desplegando las 2 Apps

Trabajaremos con las carpetas recepcion/ y alumno/.

  • Entra a js/app.js en cada carpeta y pega la URL de tu API.
  • Sube ambas carpetas a tu repositorio en GitHub.
  • Habilita GitHub Pages.

El toque final:
La app generará un Código QR genérico. Es la misma URL para todos, pero cada alumno verá sus propios datos al iniciar sesión.

¡Prueba la Integración Final! 🚀

1. Recepción: Abre recepcion/index.html y registra un alumno (verás el QR).

2. Alumno: Escanea el QR con tu celular y entra a la app con las credenciales.

3. Administrador: Modifica la nota o deuda en Sheets y mira cómo se actualiza en el celular.
